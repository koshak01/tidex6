# tidex6 — Брендбук

Векторный логотип, цвета бренда и React-компоненты для интеграции на сайт `tidex6` и в любые поверхности проекта.

Все ассеты построены под палитру Solana Foundation, к которой проект привязан (хакатон Colosseum Frontier, RPC-инфраструктура Helius/Triton One).

---

## 1. Концепция

Логотип — стилизованный котелок (bowler hat). Прочтение: «hat tip» — жест приветствия и согласия. Соответствует слогану проекта *«I grant access, not permission»*: шляпу снимают добровольно, не по принуждению.

Геометрически — два примитива:
- **тулья** (купол + прямые стенки + пологая нижняя кромка),
- **поля** (эллиптическое кольцо: внешний эллипс минус внутренний).

Конверсия из исходного PNG в SVG выполнена через подгонку эллипсов методом наименьших квадратов. **Точность совпадения с оригиналом: IoU = 98.41%** (отличия только на субпиксельных границах из-за антиалиасинга растра).

## 2. Дизайн-токены

### 2.1. Цвета

| Токен            | HEX        | RGB              | Использование                                   |
|------------------|------------|------------------|-------------------------------------------------|
| `solana-purple`  | `#9945FF`  | `153, 69, 255`   | Основной цвет бренда (моно), старт градиента   |
| `solana-green`   | `#14F195`  | `20, 241, 149`   | Акцентный цвет, конец градиента                 |
| `solana-bridge`  | `#19B4D4`  | `25, 180, 212`   | Опциональный middle-stop в 3-stop градиенте    |
| `bg-dark`        | `#0a0a0a`  | `10, 10, 10`     | Рекомендованный фон под логотип                |

CSS-переменные:
```css
:root {
  --solana-purple: #9945FF;
  --solana-green:  #14F195;
  --solana-bridge: #19B4D4;
  --bg-dark:       #0a0a0a;
}
```

### 2.2. Brand gradient

Канонический градиент Solana — двухцветный, между `--solana-purple` и `--solana-green`. RGB-интерполяция между ними естественно проходит через серо-голубую зону (так получается узнаваемый «Solana blue» на середине).

```css
background: linear-gradient(90deg, #9945FF 0%, #14F195 100%);   /* горизонтальный (по умолчанию) */
background: linear-gradient(45deg, #9945FF 0%, #14F195 100%);   /* диагональ */
```

Если на середине нужен более насыщенный цвет (особенно при диагональном направлении и больших площадях) — добавляется опциональный третий stop `#19B4D4` на 50%. Это уже отступление от двухцветного канона Solana, но визуально живее на крупных hero-блоках.

### 2.3. Геометрия логотипа

Все размеры в координатах `viewBox="0 0 624 374"`:

| Элемент                  | Тип       | Параметры                                     |
|--------------------------|-----------|-----------------------------------------------|
| Внешний контур полей     | эллипс    | `cx=312, cy=291, rx=312, ry=82`              |
| Отверстие в полях        | эллипс    | `cx=312, cy=265, rx=274, ry=59`              |
| Купол тульи              | полуэллипс| `cx=312, cy=200, rx=216, ry=200`             |
| Стенки тульи             | прямые    | `x=96, x=528, y=200..271`                     |
| Нижняя кромка тульи      | дуга      | `cx=312, cy=213, rx=300, ry=84` (нижняя ветка)|

`viewBox` сознательно нормализован до **624×374** (исходник был 625×374) — для целочисленных координат и удобной арифметики.

### 2.4. Соотношение сторон

`624 : 374 ≈ 1.668 : 1`

Высота из ширины: `height = width * 374 / 624 = width * 0.5994`.
Ширина из высоты: `width = height * 624 / 374 = height * 1.6684`.

## 3. Файлы логотипа (`logo/`)

Все версии используют идентичную геометрию и отличаются только цветовым заполнением.

| Файл                      | Размер  | Цвет / Gradient                                   | Когда использовать                                                                 |
|---------------------------|---------|---------------------------------------------------|------------------------------------------------------------------------------------|
| `hat.svg`                 | 305 B   | Моно `#9945FF`                                    | По умолчанию. Основной моно-вариант на тёмном фоне.                                |
| `hat.min.svg`             | 274 B   | Моно `#9945FF`                                    | Минифицированная версия (в одну строку). Для inline в HTML, data URL, email-шаблонов. |
| `hat-mono-green.svg`      | 305 B   | Моно `#14F195`                                    | Зелёный акцент. Для зелёных секций сайта или второстепенных применений.            |
| `hat-solana.svg`          | 533 B   | Brand gradient, **горизонтальный (90°)**          | Универсальный gradient. Чистый brand. Цвета максимально насыщенные.                |
| `hat-solana-45a.svg`      | 535 B   | Brand gradient, **45° (BL→TR)**                   | CSS-стандартный 45°. Фиолет внизу-слева, зелёный сверху-справа.                    |
| `hat-solana-45b.svg`      | 535 B   | Brand gradient, **135° (TL→BR)**                  | Обратный диагональный. Фиолет на куполе, зелёный на полях. Эффект «свет сверху».  |
| `hat-solana-45c.svg`      | 584 B   | Brand gradient, **45° + middle stop `#19B4D4`**   | 3-stop версия для hero-баннеров. Ярче за счёт насыщенной середины.                 |

### Рекомендации по применению

- **Favicon, navbar, footer, мелкие иконки (≤ 64px):** `hat.svg` или `hat.min.svg`. На малых размерах градиенты сливаются — моно работает чище.
- **Hero-баннеры, OG-image, карточки соцсетей (≥ 200px):** `hat-solana-45c.svg` или `hat-solana.svg`. Градиенты раскрываются на крупных размерах.
- **Презентации, питч-дек, hackathon submissions:** `hat-solana.svg` (горизонтальный gradient — каноничный Solana-style).
- **Печать, ч/б документы:** `hat.svg` с переопределённым `fill` через CSS.

### Inline в HTML

Содержимое `hat.min.svg` (274 байта) можно вставлять напрямую в HTML — это меньше, чем data URL или внешний запрос:
```html
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 624 374" fill="#9945FF">...</svg>
```

## 4. React-компонент (`components/`)

### 4.1. Файлы

| Файл              | Описание                                |
|-------------------|-----------------------------------------|
| `HatLogo.jsx`     | JavaScript-версия (для проектов без TS) |
| `HatLogo.tsx`     | TypeScript-версия с типизированными props |

Оба файла функционально идентичны. TS-версия экспортирует тип `HatLogoProps` для использования в родительских компонентах.

### 4.2. API

| Prop            | Тип       | Default       | Описание                                                                  |
|-----------------|-----------|---------------|---------------------------------------------------------------------------|
| `size`          | `number`  | `64`          | Ширина в пикселях. Высота вычисляется автоматически (`size * 374 / 624`). |
| `color`         | `string`  | `'#9945FF'`   | Цвет в моно-режиме. Игнорируется при `gradient=true`.                     |
| `gradient`      | `boolean` | `false`       | Включить brand gradient.                                                  |
| `gradientFrom`  | `string`  | `'#9945FF'`   | Стартовый цвет градиента.                                                 |
| `gradientTo`    | `string`  | `'#14F195'`   | Конечный цвет градиента.                                                  |
| `gradientMid`   | `string`  | `undefined`   | Опциональный средний stop (для 3-stop градиента, например `'#19B4D4'`).   |
| `angle`         | `number`  | `0`           | Угол вращения градиента: `0` — горизонталь, `45` — диагональ, `90` — вертикаль. |
| `className`     | `string`  | `''`          | CSS-класс для стилизации.                                                  |
| `...rest`       | —         | —             | Любые валидные SVG-атрибуты (`onClick`, `aria-label`, `style` и т.д.).    |

### 4.3. Примеры использования

```jsx
import HatLogo from '@/components/HatLogo';

// Базовый моно-вариант — навбар
<HatLogo size={32} />

// Hero-блок — 3-stop диагональный gradient
<HatLogo size={120} gradient angle={45} gradientMid="#19B4D4" />

// Зелёный моно-акцент
<HatLogo size={48} color="#14F195" />

// Кастомная палитра (если когда-нибудь захочется уйти от Solana)
<HatLogo size={80} gradient gradientFrom="#9945FF" gradientTo="#00D4FF" />

// С интерактивом и accessibility
<HatLogo
  size={64}
  gradient
  className="cursor-pointer hover:scale-110 transition"
  onClick={() => router.push('/')}
  aria-label="tidex6 home"
  role="img"
/>
```

### 4.4. Технические детали реализации

**Уникальный gradient-id через `React.useId()`** — гарантирует, что несколько `<HatLogo gradient />` на одной странице не конфликтуют по `<defs>`.

**Группировка через `<g fill={fill}>`** — оба `<path>` (тулья и поля) окрашиваются от bounding box всей группы. Градиент непрерывен через всю фигуру: тулья и поля выглядят как одно целое, переход цвета не разрывается на стыке элементов. Если бы fill применялся к каждому path отдельно, gradient считался бы у каждого независимо и фигура смотрелась бы рваной.

**`gradientTransform="rotate(angle 0.5 0.5)"`** — поворот градиента вокруг центра bounding box. Сам объект не трогается, меняется только направление перехода цвета. Это даёт точный контроль угла без переписывания координат `x1/y1/x2/y2`.

**`fillRule="evenodd"`** на полях — стандартный приём «эллипс минус эллипс». Внутренний эллипс автоматически становится отверстием в составе того же `<path>`.

## 6. Контекст для Claude Code

Для имплементации в `tidex6.jsx` или новый компонент:

1. Скопировать `components/HatLogo.tsx` (или `.jsx`) в проект, например в `src/components/HatLogo.tsx`.
2. Импортировать и использовать вместо текущего inline-SVG в hero-блоке.
3. Если в проекте есть Tailwind — `className` передаётся напрямую.
4. Если нужны статические SVG для OG-image, favicon, README — использовать файлы из `logo/`.

Все SVG валидны по спецификации W3C, без зависимостей, без скриптов внутри. Цвета захардкожены в файлах логотипа, но в React-компоненте управляются через props.
