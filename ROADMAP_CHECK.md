# Roadmap tidex6 — чек-лист

Пройди по пунктам и отметь статус. Потом я перепишу публичный roadmap на сайте
честно: что реально сделано — наверх (Shipped), остальное — в планы, твоими словами.

**Как отмечать:** `[x]` сделано · `[~]` частично · `[ ]` нет

---

## 1. Shielded pool (приватный пул, Groth16)
**Что это:** базовый пул. Кладёшь деньги — на цепочке виден только «отпечаток»
(commitment hash), не видно кто, кому, зачем. ZK-доказательство (Groth16) прячет
связь отправитель↔получатель.
Статус: [ x]

## 2. Immutable verifier (неизменяемая программа-проверяющий)
**Что это:** программа на Solana, что проверяет доказательства. Ключ на изменение
**отречён** — её нельзя переписать никогда, ни нами, ни хакером. OtterSec проверил
исходник.
Статус: [x ]

## 3. Relayer (релеер — вывод без следа)
**Что это:** сервис, который платит комиссию за твой вывод из своего кошелька.
Твой кошелёк получателя вообще не светится на цепочке как плательщик → аналитика
не свяжет.
Статус: [x ]

## 4. Envelope memo аудитору (зашифрованная записка проверяющему)
**Что это:** к депозиту можно приложить зашифрованную записку (дата, сумма, за что),
адресованную **аудитору** (не получателю). Только у кого есть ключ аудитора — прочтёт.
Публика не видит.
Статус: [x ]

## 5. Client-side proof (доказательство в браузере, WASM)
**Что это:** доказательство считается прямо в твоей вкладке (~1.7 сек). Твой секрет
**никогда не покидает браузер** — на сервер уходит только готовое доказательство.
Статус: [x ]

## 6. Stealth payments (скрытые платежи — получатель сам находит)
**Что это:** ноту получателю **не передаём** никак (ни файлом, ни ссылкой). Получатель
сам сканирует цепочку своим ключом и находит платёж себе. Его кошелёк не светится.
Статус: [x ]

## 7. ML-KEM-768 (пост-квантовое шифрование)
**Что это:** всё шифрование (записки, адреса) на пост-квантовом алгоритме — устоит
даже против будущего квантового компьютера («записали сейчас — не расшифруют потом»).
Статус: [x ]

## 8. Hidden amount (скрытая сумма, Token-2022 CT / wUSDC)
**Что это:** прячем не только связь, но и **саму сумму**. На цепочке сумма
зашифрована. Кладёшь **любую сумму** (хоть 1, хоть 10 — не делим на фиксированные
номиналы).
Статус: [x ]

## 9. Multi-asset (несколько монет — USDC + USDT)
**Что это:** отдельные пулы под USDC и под USDT (стейблкойны), не только SOL.
Статус: [x ]

## 10. Ceremony (публичная церемония trusted setup)
**Что это:** публичная многосторонняя генерация параметров пула. Много независимых
людей добавляют свою случайность → систему нельзя подделать, пока хоть один был честен.
Страница ceremony.tidex6.com собирает вклады.
Статус: [x ]

## 11. X25519 view-tags (быстрый скан)
**Что это:** метка-ускоритель. При сканировании цепочки получатель одной быстрой
операцией отсеивает ~255 из 256 чужих платежей, прежде чем делать дорогую расшифровку.
Скан из часов → в минуты.
Статус: [x ]

## 12. Reproducible build + /verify (воспроизводимая сборка)
**Что это:** любой может пересобрать браузерный код из исходника и убедиться, что байты
в его вкладке = открытый код (хеш совпадает). Страница /verify проверяет вживую.
Нет скрытого бэкдора.
Статус: [x ]

## 13. Wallet Standard (несколько кошельков)
**Что это:** подключение не только Phantom, а Solflare / Backpack / Ledger — через
единый стандарт, чтобы нельзя было подсунуть фейковый кошелёк с подменённым адресом.
Статус: [ x]

## 14. Security hardening (CSP, self-host библиотек)
**Что это:** защита сайта — строгая политика загрузки скриптов (CSP), все библиотеки
свои (не с чужих CDN), чтобы никто не подменил код на лету.
Статус: [x ]

## 15. tip-jar (пример интеграции через CPI)
**Что это:** сторонняя программа-пример: любой Solana-проект может подключить нашу
приватность в ~30 строк Rust. Задеплоена на mainnet, OtterSec-проверена.
Статус: [x ]

## 16. Revoke / refund (вернуть незабранный депозит)
**Что это:** если получатель так и не забрал платёж — отправитель может вернуть свои
деньги через N дней (revoke window). Механизм в SDK есть; кнопки в UI пока нет.
Статус: [x ]

## 17. Regulated / multi-auditor pools (пул с регулятором)
**Что это:** пул, где у регулятора есть ключ на просмотр **всего пула** — он видит
транзакции, но **заморозить/остановить не может**. Аудит без права стопа. Для USDT —
легальная приватность.
Статус: [ ]

## 18. Proof of Innocence / association sets (доказательство чистоты)
**Что это:** доказать «мои деньги НЕ из грязного источника (не хак, не санкции)», при
этом **не раскрывая кто я**. Пруф «я в белом списке чистых», а не «вот мой адрес».
Ответ тем, кто путает приватность с отмыванием.
Статус: [ ]

## 19. Time-scoped auditor keys (ключ аудитора на период)
**Что это:** выдать бухгалтеру ключ, что видит **только один период** (напр. только
2026 год для налоговой), а не всю историю навсегда. Сейчас ключ = «всё или ничего».
Статус: [ ]

## 20. Relayer hardening (укрепление релеера)
**Что это:** сейчас релеер — один горячий кошелёк (ключ на сервере). Укрепить: ключ в
железе (HSM), холодный мультисиг, **несколько независимых релееров** — чтобы приватность
не зависела от одного нашего сервера.
Статус: [ ]

## 21. Timing defences (защита по времени)
**Что это:** если депозит и вывод близко по времени — по таймингу можно догадаться о
связи. Защита: случайная задержка/окно → тайминг не выдаёт.
Статус: [ ]

## 22. UTXO — сдача и объединение нот
**Что это:** НЕ «произвольная сумма» (она уже есть). Это как в Bitcoin: нота на 10 —
отдать 3, а **сдача 7** вернётся тебе новой приватной нотой. Или **склеить** две ноты
в одну. Или 1 вход → несколько выходов за раз. Сейчас нота тратится целиком.
Статус: [ ]

## 23. Shared anonymity pool (общий пул анонимности)
**Что это:** сейчас у каждого приложения своя «толпа» для анонимности. Общий пул = все
приложения мешаются в одной толпе → чем больше интеграций, тем сильнее приватность у всех.
Статус: [ ]

## 24. Falcon PQ-подпись (пост-квантовая подпись)
**Что это:** пост-квантовое шифрование (ML-KEM) у нас уже есть (п.7). Falcon — это
пост-квантовая **подпись** тратящего. Закрыть последний классический примитив от кванта.
Research-спайк.
Статус: [ ]

## 25. Persistent browser prover (кеш ключа в браузере)
**Что это:** сейчас браузер каждый раз заново грузит/разбирает proving key. Закешировать
между вызовами → доказательства ещё быстрее.
Статус: [ ]
